LeetCode-两个排序后的数组求中位数

5月 27, 2018

闲来无事，刷了一道算法题还蛮有意思的。题目地址leetcode 。

题目大致要求是 给出两个排过序的升序数组，要求得这两个数组的中位数，并且要求算法复杂度至少是O(log (m+n))。

O(min(m,n)log(max(m,n)))

首先先从简单往复杂考虑，先理解怎么能求得这个中位数，在不考虑算法复杂度的情况下很简单。就把两个数组 合并成一个 升序的数组，那中位数就要么是(n是合并之后数组个数)

  1. Array[n/2] (n为奇数)
  2. (Array[n/2 - 1] + Array[n/2])/2 (n为偶数)

由于两个数组都是升序数组，所以选择长度短的那个数组，通过二分法进行插入。代码参考 cpp 。

但是这个方法没有满足 复杂度的要求，上述算法的复杂度大致是 min(m,n)log(max(m,n))。

不过居然也通过了leetcode的验证，看来它这个测试的机制还是有些欠缺。

O(log (m+n))

第二种在考虑到要满足算法复杂度的情况下，由log必然想到利用了二分法 或者 分治法处理。而 m+n 则是把两个数组统一起来去考虑。

在不合并这两个数组的情况下，在两个数组中找到第(m+n)/2大的数就可以了。

假设 k = (m+n)/2，在array1 和 array2 分别找 k/2大的位置的数。
